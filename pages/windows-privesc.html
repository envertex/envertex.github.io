Enumeration

Enumeration is not a phase.
It is a continuous feedback loop.
Every exploit, shell, or failure starts here.

Enumeration Mindset

  • No exploit without service understanding
  • No shell means incomplete enumeration
  • If stuck → enumerate again, deeper
  • Everything leaks information

Nmap – Network Enumeration

Nmap defines the entire attack surface.
Miss a port and you miss the machine. 

nmap -sC -sV -p- $IP

      
nmap -p- --min-rate 5000 -n $IP

      
nmap -sU --top-ports 1000 $IP
  • Identify all services
  • Confirm versions manually
  • Note unusual ports and protocols

HTTP / HTTPS Enumeration

Web services almost always hide something.
Enumeration must be recursive and methodical. 

whatweb http://$IP

      
gobuster dir -u http://$IP -w /usr/share/wordlists/dirb/common.txt

      
ffuf -u http://$IP/FUZZ -w /usr/share/seclists/Discovery/Web-Content/common.txt
  • View source code
  • Look for parameters and APIs
  • Test uploads and authentication

SMB Enumeration

SMB frequently leaks users, shares, and credentials. 

smbclient -L //$IP -N

      
enum4linux -a $IP
  • Anonymous access
  • Readable shares
  • Scripts and configs with creds

FTP Enumeration

ftp $IP
  • Anonymous login
  • Writable directories
  • Upload equals potential shell

SSH Enumeration

  • Version-specific vulnerabilities
  • Credential reuse potential
  • Target for brute-force later

Database Enumeration

  • MySQL / PostgreSQL / MSSQL
  • Default or reused credentials
  • File read/write primitives

Enumeration Loop

  • Service discovered → deeper enumeration
  • Exploit fails → enumerate again
  • Shell obtained → local enumeration begins
