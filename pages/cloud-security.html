Cloud Security – Attacking the Cloud

Cloud attacks are just misconfigurations, leaked credentials, and bad trust models… but at scale.

Cloud Attack Surface

  • Public cloud services exposed to the internet
  • Leaked API keys and access tokens
  • Misconfigured IAM policies
  • Insecure storage services
  • Trust relationships between services

Credential Exposure in Cloud

  • Environment variables in applications
  • Source code repositories
  • CI/CD pipelines
  • Metadata services
  • Backup files and logs

Cloud Enumeration

  • Identifying cloud provider (AWS, Azure, GCP)
  • Enumerating permissions with stolen credentials
  • Listing storage buckets and containers
  • Discovering exposed services

Storage Attacks

  • Public buckets and containers
  • Read/write misconfigurations
  • Credential harvesting from backups
  • Uploading malicious files

IAM Abuse

  • Privilege escalation via overly permissive roles
  • Assuming roles across services
  • Creating new access keys
  • Persistence through IAM backdoors

Metadata Service Attacks

  • Accessing cloud metadata endpoints
  • Stealing temporary credentials
  • Pivoting from VM to cloud account

Lateral Movement in Cloud

  • Service-to-service trust abuse
  • Compromised VM → cloud API
  • Cloud credentials reused on on-prem AD

Persistence in Cloud

  • New access keys
  • Backdoored IAM roles
  • Malicious automation scripts
  • Hidden scheduled jobs

Cloud and Active Directory

  • Hybrid environments (AD + Cloud)
  • Azure AD trust abuse
  • Password reuse between cloud and on-prem

Cloud Attack Mindset

  • Cloud is just another network
  • Identity is the new perimeter
  • Misconfigurations are king
  • Credentials > exploits
