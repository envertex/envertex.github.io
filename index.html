OSCP Attack Methodology

This repository is a mental framework for the OSCP exam. It is not about memorizing commands, but about knowing what to do next when something fails.

Every machine gives feedback. Ports, services, errors, credentials — all of them are signals.

If you feel lost during the exam, return here and reset your thinking.

Standalone Machine (Non-AD)

This diagram represents a typical standalone OSCP machine. The objective is always the same: Initial access → Local enumeration → Privilege escalation.

Enumeration is not a single phase. It is a loop that happens before exploitation and again after gaining a shell.

  • If no exploit works → enumeration is incomplete
  • If you get a shell but no privesc → local enumeration failed
  • If everything looks patched → re-check services and versions

Never jump steps. When stuck, move backwards in the diagram and re-evaluate assumptions.

Active Directory Environment

Active Directory machines require a different mindset. Exploits matter less than credentials, permissions, and trust relationships.

Progress in AD is rarely linear. Small wins (like one user credential) often unlock multiple attack paths.

  • User access is more important than SYSTEM shells
  • Domain enumeration never truly ends
  • Lateral movement is expected, not optional

If one attack path fails, enumerate again and search for another. Domain compromise is about persistence, not speed.

Exam Mindset

  • Slow thinking beats rushed exploitation
  • Trust enumeration over gut feeling
  • Document everything, even failed attempts
  • If stuck for too long, reset and re-enumerate
