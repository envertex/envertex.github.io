Path Traversal

Path Traversal (also known as Directory Traversal) is a vulnerability that allows an attacker to access files and directories outside of the intended web root. Although it may seem simple, exploiting Path Traversal requires understanding how applications process file paths, validations, and encodings.

The goal of this attack is to read arbitrary files that the application was not designed to expose, such as system configuration files, credentials, or source code.

Vulnerable Parameter

Consider the following example: 

https://0a1b0073032c7a1f80dd993c0049001e.web-security-academy.net/image?filename=5.jpg

In this URL, the filename parameter is used by the backend to locate and return a file. If proper validation is not implemented, this parameter may be vulnerable to Path Traversal.

Basic Exploitation

To prove arbitrary file read, the attacker commonly attempts to access sensitive files such as /etc/passwd on Linux systems: 

filename=/etc/passwd
filename=../../../etc/passwd

The traversal sequences (../) allow navigating up the directory hierarchy until reaching the root directory.

Encoding Bypasses

Applications often block raw traversal sequences. To bypass these filters, attackers can use URL encoding or double URL encoding: 

..%2F..%2F..%2Fetc%2Fpasswd
..%252F..%252F..%252Fetc%252Fpasswd

URL decoding may occur multiple times on the server, allowing double-encoded payloads to bypass validation.

Absolute Path Validation Bypass

In some cases, the application constructs the file path internally, for example: 

/var/www/images/

Even if the application validates that the supplied path starts with a specific directory, traversal can still occur: 

/var/www/images/../../../etc/passwd

This works because the operating system resolves the traversal sequences after path concatenation.

Null Byte Injection

Some applications append a file extension to the user input (e.g. .png). In legacy systems, a null byte (%00) can terminate the string: 

../../../etc/passwd%00.png

This technique is effective in older languages or improperly handled file APIs.
